=== KEY BEHAVIORAL DISCOVERIES ===

pattern_sequencer "first trigger wasted":
  - IDLE state output logic: null (trigger_out='0')
  - RUNNING state output logic: trigger_out = trigger_in (combinational)
  - First trigger: IDLE->RUNNING, loads table[0], NO trigger_out
  - Second trigger: RUNNING, outputs table[0], loads table[1], trigger_out='1'
  - N entries -> N-1 trigger_outs before wrap/done

timing_controller 1-cycle trigger_out:
  - trigger_out is combinational: '1' only when current_state=FIRED
  - FIRED state lasts exactly 1 clock cycle
  - Must use "wait until trigger_out='1'" not fixed wait

trigger_mux path delays:
  - TTL: 3 cycles (sync_1 at cycle 1, sync_2+ttl_edge at cycle 2, output_reg at cycle 3)
  - USB: 1 cycle (usb_edge combinational, output_reg at cycle 1)
  - Timer: 1 cycle (same as USB)
  - trigger_count increments on trigger_fired (1 cycle before trigger_out)
